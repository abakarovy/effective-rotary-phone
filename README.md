# KBC Test Script

Скрипт открывает Chrome через Selenium, ждёт ручного входа, затем по данным из Excel открывает видео во вкладке и отправляет ответы на задания через API (форматы 3.1 и 3.2).

## Установка

```bash
pip install -r requirements.txt
```

Нужен Chrome и ChromeDriver (или Selenium 4 с автоматической загрузкой драйвера).

## Запуск

1. Положите `test1.xlsx` и `test2.xlsx` в корень проекта (или укажите пути в переменных окружения).
2. (Опционально) Создайте файл `videos.txt` — по одной ссылке на видео в строке. Сразу после входа все ссылки откроются в новых вкладках первого окна.
3. Запустите:

```bash
python run.py
```

3. В открывшемся окне Chrome войдите на сайт.
4. В терминале нажмите Enter — скрипт откроет все ссылки из `videos.txt` в **новых вкладках** того же окна, затем выполнит 3.1 и 3.2 в том же браузере.

## Конфиг

- `KBC_BASE_URL` — базовый URL (по умолчанию `https://kb.cifrium.ru`).
- `KBC_EXCEL_31`, `KBC_EXCEL_32` — пути к `test1.xlsx` и `test2.xlsx`.
- `KBC_LOGIN_WAIT_SEC` — сколько ждать до нажатия Enter после открытия страницы (по умолчанию 60).
- `KBC_ANSWER_SIMILARITY` — порог совпадения ответа из Excel с вариантом в HTML для радио (0.7).
- `KBC_HEADLESS` — `1`/`true` для запуска без окна браузера.
- `KBC_VIDEO_LINKS_FILE` — путь к файлу со ссылками на видео (по умолчанию `videos.txt`). Одна ссылка на строку; все открываются в новых вкладках сразу после входа.

## Excel

- Без заголовков. Колонки: id задания (classwork_id для 3.1, lesson_id для 3.2), id вопроса/задачи, ответ текстом.
- Для радио ответ из Excel сопоставляется с вариантами на странице по схожести текста; для текстовых полей подставляется как есть.
